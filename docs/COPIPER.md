# CoPiper 配表工具

CoPiper 是游戏配置数据（JDB 格式）的电子表格编辑器和导出工具，支持复杂的类型系统、跨文件引用解析和 Python/JSON 导出。

## 功能概览

- JDB 文件浏览：递归扫描工作区 `.jdb` 文件，树形展示数据库→表层级
- 表结构管理：增删改表、列定义编辑（类型、名称、默认值、引用源等）
- 数据编辑：基于 Handsontable 的电子表格式编辑，支持所有列类型的专用编辑器
- 数据验证：必填检查、ID 唯一性、类型合规、引用完整性
- 数据导出：Python (.py) + JSON 格式，完整类型解析管线
- 跨文件引用：自动从工作区所有 JDB 文件加载引用表数据
- 脚本集成：支持自定义检查脚本和后处理脚本

## 架构

```
Renderer (React + Zustand)
  ├─ CopiperPage (布局)
  ├─ CopiperSidebar (文件/表树)
  ├─ CopiperToolbar (操作按钮)
  ├─ CopiperTable (Handsontable)
  └─ ExportModal / ValidationPanel / ColumnEditor / RowDetailDrawer

  ↓ window.api.copiper.* (IPC)

Main Process (Node.js)
  ├─ copiper.ipc.ts (17 个 IPC 处理器)
  ├─ copiper.service.ts (业务逻辑 + 导出引擎)
  ├─ jdb.service.ts (JDB 文件 I/O)
  └─ copiper.store.ts (electron-store 持久化)

  ↓ 文件系统

workspace/data/**/*.jdb (JDB 数据文件)
workspace/game/common/data/**/*.py|.json (导出目标)
workspace/data/copiper/check/*.py (检查脚本)
workspace/data/copiper/post_process/*.py (后处理脚本)
```

## 数据模型

### JDB 文件结构

```typescript
// 整个 JDB 文件
type JDBDatabase = Record<string, JDBTableData>

// 单张表
interface JDBTableData {
  columns: ColDef[]
  rows: RowData[]
}
```

### 列定义 (ColDef)

| 字段 | 说明 |
|------|------|
| `name` | 字段名（数据 key） |
| `rname` | 显示名称（UI 展示） |
| `type` | 原始类型，如 `"index/ResourceData"`, `"list:kv:item_list"` |
| `j_type` | 统一类型，如 `"index"`, `"list:kv:item_list"`, `"utc_time:+8"` |
| `c_type` | 列类别：`data`（导出）/ `sup`（辅助）/ `rdesc`（注释） |
| `req_or_opt` | `"required"` / `"optional"` |
| `default_v` | 默认值 |
| `src` | 引用源表名（index/indices 类型） |
| `options` | 枚举选项（管道分隔或数组） |
| `is_key` | 是否主键（kv 列表转 keyed dict 用） |

### 行数据 (RowData)

| 字段 | 说明 |
|------|------|
| `id` | 主键 |
| `idx_name` | 索引名称（被其他表引用时使用） |
| `_should_export` | 是否导出（系统字段，`c_type='sup'`） |
| `_deprecated` | 是否废弃 |
| `[key]` | 动态字段 |

## 列类型系统

### 编辑器

| 类型 | 编辑器 | 说明 |
|------|--------|------|
| `str` | 文本输入 | 字符串 |
| `int` | 数字输入 | 整数，支持十六进制 (0x) |
| `float` | 数字输入 | 浮点数 |
| `bool` | 复选框 | 布尔值 |
| `enum` | 下拉选择 | 枚举，选项来自 `options` 字段 |
| `index/Table` | 下拉选择 | 单引用，选项来自源表 idx_name |
| `indices/Table` | 下拉选择 | 多引用（`j_type: "list:index"`），管道分隔 |
| `kv:Name` | JSON Modal | 键值对，按结构定义递归解析 |
| `ckv:Name` | JSON Modal | 条件键值对，含 `cls` 类名字段 |
| `list:kv:Name` | JSON Modal | KV 列表，可转 keyed dict |
| `list:ckv` | JSON Modal | CKV 列表 |
| `list:*` | 文本 | 管道分隔列表，按子类型解析 |
| `utc_time` | 日期选择 | UTC 时间戳 |
| `utc_time:+8` | 日期选择 | 带时区的时间戳 |
| `tstr` | 文本 | 模板字符串 |
| `istr` | 文本 | 索引字符串 `<Table>[idx_name]` |
| `dict` | JSON Modal | 字典 |

### 导出类型解析

导出时 `parseValue()` 按 `j_type` 分发到对应解析器，处理顺序：

```
list:kv:* → kv:* → list:ckv → list:* → ckv → index → istr → int/float/bool/str → utc_time → tstr
```

每个解析器可递归调用 `parseValue()` 处理嵌套子字段。

**ExportContext** 预构建以下映射：
- `srcFromToMap`: 表名 → { idx_name → id } — 用于 index 引用解析
- `srcToFromMap`: 表名 → { id → idx_name } — 用于反向查找
- `allColsInfo`: 表名 → { 列名 → ColDef } — 用于 kv/ckv 结构字段查找

### 跨文件引用

Index 列（如 `index/ResourceData`）引用的表可能在不同的 JDB 文件中：

- **导出时**：`loadAllWorkspaceTables()` 扫描工作区所有 `.jdb` 文件，合并为完整的 `allTables`
- **编辑时**：`loadReferenceData()` 按需加载引用表的 `{ id, idx_name }` 数据，填充下拉选项

## 导出管线

### 单表导出 (exportTable)

```
1. 加载跨文件引用表
2. buildExportRows() — 类型解析 + 引用解析
3. 空表检测 → 跳过 (skipped: true)
4. 运行检查脚本 (data/copiper/check/{relDir}_{tableName}.py)
5. 检查失败 → 中止
6. buildDefaultData() — 构建 DefaultData
7. 生成文件：Python (.py) 或 JSON
8. 运行后处理脚本
```

### 批量导出 (exportAll)

```
Phase 1: 逐表导出（跳过后处理）
Phase 2: 运行 all.py 后处理（一次，处理所有表）
Phase 3: 逐表运行后处理脚本（空表标记 Skipped）
```

### Python 输出格式

```python
# Auto generated by CoPiper, do not edit it.

# 数据来源: basic/11_道具表_ItemData.xlsx, ItemData

DefaultData = {
    "id": "default",
    "name": "",
    "count": 0
}

DataList = [
    {"id": 10001, "name": "\u82f9\u679c", "count": 100},
    {"id": 10002, "name": "\u9999\u8549", "count": 50}
]
```

### 导出路径

- JDB 位于：`data/{relDir}/{filename}.jdb`
- Python 输出到：`game/common/data/{relDir}/{tableName}.py`
- JSON 输出到：`game/common/data/{relDir}/{tableName}.json`

## 脚本集成

### 检查脚本

路径：`data/copiper/check/{relDir}_{tableName}.py`

在导出前运行，输出 JSON：
```json
{"result": true, "info": "检查通过", "r_c_e": []}
```

`result: false` 时中止导出。

### 后处理脚本

路径：`data/copiper/post_process/{relDir}_{tableName}.py` 或 `data/copiper/post_process/all.py`

在导出文件写入后运行。`all.py` 对所有表执行一次，表级脚本逐表执行。

### 平台兼容

- **xlog 存根**：自动生成 no-op `xlog.py` 模块（原始脚本依赖的 C 扩展不可用）
- **路径兼容**：`_compat_runner.py` 包装器修补 `os.path.join` 处理 Windows 反斜杠路径

PYTHONPATH 包含：xlog 存根目录、`data/`、`data/xconvertor/`、`data/copiper/`

## 验证

`validateTable()` 检查：

| 规则 | 级别 |
|------|------|
| ID 唯一性 | error |
| 必填字段非空 | error |
| int 列值合法 | error |
| float 列值合法 | error |
| index 引用存在 | warning |
| indices 引用存在 | warning |

## 状态管理 (useCopiperStore)

| 状态 | 说明 |
|------|------|
| `databases` | JDB 文件列表 |
| `activeFilePath` | 当前打开的 JDB 路径 |
| `activeDatabase` | 当前 JDB 内存数据 |
| `activeTableName` | 当前选中表名 |
| `referenceData` | 跨文件引用数据（index 下拉选项） |
| `dirty` | 是否有未保存修改 |
| `validationIssues` | 验证问题列表 |
| `searchText` | 搜索过滤文本 |

编辑模式：本地优先 — `updateCell()` 修改内存 + 标记 dirty，显式保存时才写入文件。Handsontable 原地修改行数据，`markDirty()` 标记状态。

## 团队协作

### 代码审查
- 所有 PR 需要至少一人审查
- 关键功能需要两人审查
- 测试必须通过才能合并

### 分支策略
- `master`: 生产分支
- `develop`: 开发分支
- `feature/*`: 功能分支
- `bugfix/*`: 修复分支

### 发布流程
1. 功能开发在 `feature/*` 分支
2. 合并到 `develop` 分支测试
3. 测试通过后合并到 `master`
4. 打 tag 发布版本
